# 데일리 이슈 — AI 역할 기반 관점 생성 (스키마·프롬프트)

운영/백엔드에서 LLM으로 이슈를 채울 때 사용하는 계약입니다. **문장을 진보/보수로 “판별”하지 않습니다.** 모델은 처음부터 네 **역할**에 맞는 한 줄 관점만 쓰고, 앱은 `choice.type`만 보고 내부 가치 벡터(`lean`)를 붙입니다.

## JSON 출력 (유일 형식)

모델은 **아래 JSON만** 반환해야 합니다 (부가 텍스트·마크다운 금지).

```json
{
  "question": "…",
  "choices": [
    { "type": "progressive", "label": "…" },
    { "type": "centrist", "label": "…" },
    { "type": "conservative", "label": "…" },
    { "type": "unsure", "label": "…" }
  ]
}
```

- `choices`는 **정확히 4개**, `type`은 위 네 값 각 **한 번씩**만 허용.
- 풀/번들에 저장할 때는 `id`를 생략해도 되며, 클라이언트가 `type`을 `id`로 맞춥니다.

## 질문 생성 규칙

- 열린 질문, **가치 충돌형**.
- 특정 정치 진영 유도·공격적 표현·정답 유도 금지.

## 선택지(label) 규칙

각 `label`은 한 역할의 **가치 우선순위**를 드러내야 합니다.

| type | 역할(내부 이름만, UI 비노출) | label 쪽 성격 |
|------|------------------------------|----------------|
| `progressive` | 지원·공정·확대 쪽 가치 우선 | 예: 지원 확대, 취약층 우선 |
| `centrist` | 균형·절충 | 예: 속도와 안정 함께 |
| `conservative` | 재정·질서·지속 가능성 | 예: 부담·지속성 함께 짚기 |
| `unsure` | 유보·판단 보류 | 예: 정보 부족, 더 지켜보기 |

**금지(라벨 문구에 넣지 말 것):** “진보에 가깝다”, “보수적”, “좌파”, “우파”, “진영” 등 **정치 진영 단어**.

**허용:** 구체적 가치·우선순위 표현(위 표 성격).

## 내부 매핑 (클라이언트)

`public/alignment-scoring.js`의 `DAILY_ISSUE_ROLE_LEAN` / `leanForDailyIssueRoleType`이 `type` → `{ progressive, centrist, conservative }` 분포를 고정합니다. AI는 **lean을 출력하지 않습니다.**

## UI 쪽 참고

- 관점 버튼 **표시 순서**는 매 렌더마다 **무작위 셔플**(저장 순서와 무관).
- 유저에게는 **라벨만** 노출; `type`은 절대 노출하지 않음.

자세한 가중·댓글 게이트는 `docs/DAILY_ISSUE_CONTENT_GRAVITY.md`를 참고하세요.
