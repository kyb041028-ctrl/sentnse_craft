# 센텐스크래프트 × Supabase — 사장님용 쉬운 연동 가이드

이 문서는 **비전공자**도 따라 할 수 있게, **순서대로만** 적었습니다.  
한 단계씩 천천히 진행하면 됩니다.

---

## 0. 왜 이렇게 하나요?

- **회원가입·로그인**은 Supabase라는 **클라우드 서비스**가 처리합니다.
- 우리 **컴퓨터에서 돌아가는 서버**(`npm start`)가 Supabase와 통신합니다.
- **주소(URL)와 키(Key)**는 비밀이라서, **소스코드가 아니라 `.env` 파일**에만 적습니다.

> **중요:** 웹페이지 파일(`public/index.html`)을 탐색기에서 더블클릭해서 열면 **로그인 안 됩니다.**  
> 반드시 아래 가이드대로 **`http://localhost:3000`** 으로 접속하세요.

---

## 1. Supabase 계정 만들기

1. 브라우저에서 [https://supabase.com](https://supabase.com) 을 엽니다.
2. **Sign up / Log in** 으로 계정을 만듭니다. (GitHub로 가입해도 됩니다.)

---

## 2. 새 프로젝트(데이터 집) 만들기

1. 로그인 후 **New project** 를 누릅니다.
2. **이름**을 정합니다. (예: `sentence-craft`)
3. **데이터베이스 비밀번호**를 정합니다. **꼭 메모**해 두세요. (나중에 SQL만 돌릴 때 필요할 수 있습니다.)
4. **리전(Region)** 은 가까운 곳(예: Northeast Asia)을 고르면 됩니다.
5. **Create new project** 를 누르고, **1~2분** 정도 기다립니다. (초록불이 들어오면 준비 완료)

---

## 3. 테이블 설계(SQL) 넣기 — 한 번만 하면 됩니다

우리가 미리 만든 설계 파일이 있습니다.

- 폴더 안 경로: `supabase/schema_profiles_identity_history.sql`

**Supabase 웹 화면에서:**

1. 왼쪽 메뉴에서 **SQL Editor** 를 클릭합니다.
2. **New query** 를 누릅니다.
3. 컴퓨터에서 `schema_profiles_identity_history.sql` 파일을 메모장으로 열어 **전체 복사**합니다.
4. Supabase의 큰 입력창에 **붙여넣기** 합니다.
5. 오른쪽 아래 **Run** (또는 **실행**) 버튼을 누릅니다.
6. **Success** 같은 성공 메시지가 나오면 OK입니다.

> 이미 비슷한 테이블이 있어서 에러가 나면, 개발자에게 스크린샷을 보여주세요.

---

## 4. 이메일로 가입·로그인 켜기 (기본 설정 확인)

1. 왼쪽 메뉴 **Authentication** 을 클릭합니다.
2. **Providers** (또는 **Sign In / Providers**)에서 **Email** 이 켜져 있는지 확인합니다. (보통 기본 ON)

### 이메일 인증(확인 메일)을 켜 둔 경우

- 회원가입 직후에는 **바로 로그인이 안 될 수 있습니다.**
- 메일함의 **인증 링크**를 누른 뒤, 화면에서 **로그인**을 하면 됩니다.

---

## 5. 주소(URL)와 키(Key) — 어디서 복사하나요?

Supabase 화면 **왼쪽 아래 톱니바퀴** → **Project Settings** 를 클릭합니다.  
그다음 왼쪽에서 **API** 메뉴를 클릭합니다.

여기서 **두 가지**를 복사합니다.

| 이름 (영문) | 한글로 이해하기 | 복사해서 쓸 이름 (우리 `.env`) |
|-------------|----------------|-------------------------------|
| **Project URL** | 우리 앱이 찾아갈 Supabase 집 주소 | `SUPABASE_URL` |https://vmpdddcdnxeavchgivcd.supabase.co/rest/v1/
| **anon public** 키 (긴 `eyJ...` 문자열) | 브라우저/서버가 쓰는 **공개용** 열쇠 | `SUPABASE_ANON_KEY` |

> **주의:** `service_role` 이라고 적힌 **비밀 열쇠**는 이 프로젝트의 `.env`에 **넣지 마세요.**  
> 유출되면 누구나 DB를 마음대로 할 수 있어서 매우 위험합니다.

## 6. 내 컴퓨터에 `.env` 파일 만들기

1. 프로젝트 폴더(예: `sentence-craft`)를 엽니다.
2. `.env.example` 파일을 **복사**합니다.
3. 복사한 파일 이름을 **`.env`** 로 바꿉니다.  
   - Windows에서는 메모장으로 열 때 **이름이 `.env.txt` 가 되지 않게** 주의하세요.  
   - 탐색기 **보기 → 파일 확장명** 체크 후 이름을 `.env` 로 맞추면 편합니다.

4. `.env` 를 메모장으로 열고 아래처럼 채웁니다.

```env
PORT=3000

SUPABASE_URL=여기에 Project URL 붙여넣기

SUPABASE_ANON_KEY=여기에 anon public 키 붙여넣기
```

- `=` 앞뒤로 **빈칸 없이** 붙여도 되고, 한 칸 띄워도 됩니다.
- 값에 **따옴표**는 보통 필요 없습니다.

5. 저장합니다.

---

## 7. 서버 실행하고 화면 열기

1. **터미널**을 프로젝트 폴더에서 엽니다. (VS Code / Cursor 터미널도 OK)
2. 아래를 **순서대로** 입력합니다.

```bash
npm install
npm start
```

3. 터미널에 `http://localhost:3000/` 같은 주소가 나오면, 브라우저 주소창에 **그대로** 입력합니다.

4. 화면에서 **회원가입** → **로그인**을 시도합니다.

---

## 8. 잘 안 될 때 체크리스트

| 증상 | 확인할 것 |
|------|-----------|
| "서버에 연결할 수 없습니다" | `npm start` 가 켜져 있는지, 주소가 **`http://localhost:3000`** 인지 (파일 직접 열기 X) |
| 가입 후 바로 로그인 안 됨 | Supabase **이메일 인증** 켜짐 → 메일함 확인 |
| "Supabase 미설정" / 503 | `.env` 파일 이름·위치, `SUPABASE_URL` / `SUPABASE_ANON_KEY` 오타, 서버 **재시작** |
| 로그인은 되는데 프로필이 이상함 | SQL 스크립트가 **성공적으로** 실행되었는지 |

---

## 9. 정리

1. Supabase에서 **프로젝트**를 만든다.  
2. **SQL Editor**에 `schema_profiles_identity_history.sql`을 넣고 실행한다.  
3. **Project Settings → API**에서 **URL**과 **anon 키**를 복사한다.  
4. 프로젝트 폴더의 **`.env`**에 붙여넣는다.  
5. **`npm install`** → **`npm start`** → 브라우저로 **`http://localhost:3000`** 연다.

이 순서면 **진짜 회원가입·로그인**까지 연결됩니다.
